# TikTok Content Rules (Must Follow)

TikTokのコンテンツ（スライド＆キャプション）を生成する際は、常に以下のルールを厳守すること。

## 1. キャプションのトーン＆マナー (Caption Style)
- **ターゲット層**：日本の大学生〜20代のカップル・友達同士。
- **文体**：親しみやすく、少し感情がこもったエモい語り口調。硬い説明文（「〜は〜で有名です」）はNG。
- **構成案**：
  1. 惹きつける1行目（例：「〇〇って〇〇だけじゃないの？」）
  2. 結論（例：「実は〇〇が最強なんだよね。」）
  3. 各店舗の紹介（5〜6店舗）
     - 店名
     - 評価（⭐）と口コミ数、予算（💰）
     - 短いキャッチコピー
     - 裏情報や行くべき理由（「〇時が狙い目」「実は〇〇が人気」など行動を促す情報）
  4. 締めとCTA（「保存して次のデートに使って📌」）
  5. 関連ハッシュタグ（15〜20個。エリア名、ジャンル名、#TikTok旅行 などを含む）

## 2. スライド画像の選定基準 (Image Selection by Top Designer)
- **【デザイナーとしての絶対ルール】**：低画質（ぼやけている、ノイズが多い）、文字ばかりのメニュー表、素人が撮った暗い写真は**絶対に使用しない。**
- **検索のコツ**：`[スポット名] + 高画質 一眼レフ` `[スポット名] + 内観 プロ` `[スポット名] + 看板メニュー 綺麗` のように検索クエリを工夫し、高解像度のURL画像を優先して取得する。
- 小さなサムネイルや、Base64の低解像度プレビューで妥協しないこと。

## 3. 情報量とテキストルール (Information Density by Top Marketer)
- **【マーケターとしての絶対ルール】**：「詳細は動画でチェック」のような手抜きテキスト（ハードコード）は**絶対に禁止**。
- `batch_areas.json` 等に用意された「メリット（merits）」「デメリット（demerit）」「裏情報（secret）」を、1文字残らずスライドとキャプションに流し込むこと。
- ユーザーに「行ってみたい」「保存しておきたい」と思わせる情報密度を必ず満たすこと。

## 4. ファイル出力・構成のルール (File System Constraint)
- **デスクトップ直下への出力禁止**。
- 一連の作業で生成した一時ファイル（HTML等）は必ず削除し、ゴミを残さないこと。
- 納品物である `.zip` ファイルは、**必ず `c:\Users\taiyo_imamoto\Desktop\getmoney\ready_to_post_zips\` 内に出力**すること。
- ZIPファイル名は `[エリア名]_TikTok投稿セット_完成版.zip` で統一する。

## 4. 作業の自動化と自律性 (Autonomy)
- リサーチからZIP生成まで、人間の確認（「これでいいですか？」）をステップごとに挟まないこと。
- 一気に最後まで走り切り、納品が完了した時点でのみ報告する。
- 途中でエラー（画像の取得失敗、ZIP作成エラーなど）が起きた場合は自律的にリトライ・代替手段を実行し、それでもダメな場合のみ報告する。
